# 记忆功能测试数据

这份数据用于手工验证当前项目的记忆实现是否生效，覆盖：

- 短期记忆写入
- 历史摘要压缩
- 长期记忆事实卡片写入
- 长期记忆检索注入
- 事实去重
- AI 档案条件刷新

## 测试前准备

建议使用一个全新的 bot 名称，例如 `memory-e2e-bot`，避免历史数据干扰。

需要观察的文件：

- `data/<bot>/memory/short_term_memory.json`
- `data/<bot>/memory/history_summary.txt`
- `data/<bot>/memory/vector_store.json`
- `data/<bot>/aiprofile.md`

建议先删除旧测试数据目录：

```powershell
Remove-Item -Recurse -Force data\memory-e2e-bot -ErrorAction SilentlyContinue
```

## Case 1: 长期记忆写入

目的：验证稳定事实会被提取并写入 `vector_store.json`。

输入顺序：

1. `我叫阿舟。`
2. `我喜欢美式咖啡。`
3. `不要叫我全名，叫我阿舟。`
4. `我们现在是恋人关系。`

预期结果：

- `data/memory-e2e-bot/memory/vector_store.json` 被创建。
- 长期记忆里应至少沉淀出以下几类事实中的大部分：
  - 用户称呼/身份：`我叫阿舟`
  - 用户偏好：`喜欢美式咖啡`
  - 约束：`不要叫我全名`
  - 关系：`我们现在是恋人关系`
- 因为出现了 `constraint` 或 `relationship` 类事实，本轮后应触发一次 AI 档案刷新。

重点观察：

- 日志里应出现“长期记忆写入”相关信息。
- `data/memory-e2e-bot/aiprofile.md` 的修改时间应更新。

## Case 2: 长期记忆检索

目的：验证命中 cue 的输入会触发长期记忆检索，并把结果作为 `【长期记忆】` 注入。

输入顺序：

1. `你还记得我喜欢什么吗？`
2. `你记得该怎么称呼我吗？`
3. `你还记得我们的关系吗？`

预期结果：

- 日志里应出现长期记忆检索命中。
- 回复中应至少正确提到以下信息中的 1 到 2 项：
  - 喜欢美式咖啡
  - 称呼阿舟
  - 恋人关系

备注：

- 这里不要求回复文本逐字匹配，只看语义是否利用了之前写入的长期记忆。

## Case 3: 普通闲聊不触发长期记忆检索

目的：验证非 cue 输入不会默认查长期记忆。

输入顺序：

1. `你好啊`
2. `嗯？`
3. `在吗`

预期结果：

- 不应出现长期记忆检索命中日志。
- 即使 `vector_store.json` 已存在，这几轮也不应因为普通寒暄自动检索长期记忆。

## Case 4: 重复事实去重

目的：验证重复表达同一事实时，不会无限重复写入长期记忆。

输入顺序：

1. `我喜欢美式咖啡。`
2. `我最喜欢喝美式。`
3. `还是那句，我喜欢美式咖啡。`

预期结果：

- 长期记忆中关于“喜欢美式咖啡”的事实最多保留 1 条主事实，或者只做同一 `factKey` 的覆盖更新。
- 后续再问 `你还记得我喜欢什么吗？` 时，不应出现多条近义重复记忆同时被重复说出。

重点观察：

- `vector_store.json` 会更新，但不应因为同一事实反复输入而明显膨胀。

## Case 5: 摘要压缩触发

目的：验证短期记忆超过阈值后，会把旧对话压入摘要，并保留最近窗口。

输入顺序：

1. `第一轮：我叫阿舟。`
2. `第二轮：我喜欢美式咖啡。`
3. `第三轮：不要叫我全名。`
4. `第四轮：我们是恋人关系。`
5. `第五轮：我讨厌太甜的饮料。`
6. `第六轮：你说话可以直接一点。`
7. `第七轮：你还记得我的偏好吗？`

预期结果：

- 第 7 轮回复完成后，应触发一次摘要压缩。
- `data/memory-e2e-bot/memory/history_summary.txt` 不再为空。
- `data/memory-e2e-bot/memory/short_term_memory.json` 中应主要保留最近 4 轮对话，不再保留最早 3 轮的原始短期消息。

说明：

- 当前实现是“6 轮触发、保留最近 4 轮”的近似策略，本质上按有效对话消息数计算。

## Case 6: AI 档案 6 回合兜底刷新

目的：验证即使没有关系/约束类新事实，连续 6 个完成回合后也会触发一次档案刷新。

输入顺序：

1. `我喜欢蓝色。`
2. `我喜欢下雨天。`
3. `我喜欢篮球。`
4. `我喜欢安静一点的环境。`
5. `我喜欢晚睡。`
6. `我喜欢轻一点的音乐。`

预期结果：

- 前 1 到 5 轮不一定刷新 AI 档案。
- 第 6 轮结束后，应因兜底策略触发一次 AI 档案刷新。
- `data/memory-e2e-bot/aiprofile.md` 修改时间应更新。

## Case 7: 重启后记忆持久化

目的：验证记忆文件会在下次启动后继续生效。

步骤：

1. 先完成 Case 1 和 Case 2。
2. 关闭程序。
3. 重新启动同一个 bot：`memory-e2e-bot`。
4. 输入：`你还记得我喜欢什么吗？`

预期结果：

- 不依赖本次运行时上下文，也能重新命中长期记忆。
- `history_summary.txt`、`short_term_memory.json`、`vector_store.json` 会被重新加载。

4. Case 4
5. Case 5
6. Case 6
7. Case 7

## 快速判定标准

如果下面 5 条都成立，可以认为“记忆功能已经基本实现”：

1. 新事实能写入长期记忆。
2. 命中 cue 时能检索长期记忆。
3. 普通寒暄不会乱查长期记忆。
4. 历史对话会被摘要压缩。
5. AI 档案不是每轮刷新，但会按事件或兜底策略刷新。
